# Screenshots Checklist

Use this checklist when preparing the final submission assets.

## Required Screenshots

- [ ] Homepage hero
  - Show project title and core research positioning.
  - Include the loop: raw signal -> evidence -> risk flags -> thesis -> human decision.

- [ ] Homepage safety label
  - Capture: "Research console only. Not a buy/sell/execution system."

- [ ] Signal Inbox
  - Show four demo signals.
  - Include source, chain, category, status, Mantle relevance, risk severity, latest decision, and evidence count.

- [ ] Signal Detail top summary
  - Show signal name, source, chain, category, status, detected time, score, relevance, evidence count, and latest decision.

- [ ] Evidence panel
  - Show evidence items and confidence badges.

- [ ] Risk flags
  - Show `not_a_buy_signal` and missing-data risk flags.

- [ ] AI Research Note
  - Show thesis, why surfaced, risk summary, and next manual checks.

- [ ] Decision Trail
  - Show recorded actor, decision, reason, and timestamp.

## Optional Supporting Screenshots

- [ ] README quick start commands
- [ ] terminal output from `pnpm test`
- [ ] terminal output from `pnpm typecheck`
- [ ] terminal output from `pnpm build`

## Quality Checks

- [ ] Use laptop-width viewport.
- [ ] Avoid screenshots that crop out safety framing.
- [ ] Do not show unrelated local folders, secrets, keys, or production paths.
- [ ] Do not imply trading execution or wallet activity.
