# README — for judges

## What this is (5 lines)

1. A self-hosted research console for the Mantle Research Challenge.
2. It turns noisy Web3 scanner output into structured research objects.
3. The agent writes validated, append-only events to a JSONL store.
4. The dashboard materializes state from that store — no live trading or wallets.
5. Safety boundary: read-only, no keys, no signing, no execution.

## Hard safety boundary (verbatim from the README)

This repository is intentionally read-only and watch-only:

- no wallet connection
- no private keys
- no transaction signing
- no buy / sell / trade buttons
- no automated execution
- no Telegram live sender
- no cron jobs
- no mutation of production Hermes or Fast DEX runtime

A signal is not financial advice and not an execution instruction.

## 60-second local run

```bash
git clone https://github.com/KirillNedoboy/mantle-signal-to-thesis-console
cd mantle-signal-to-thesis-console
pnpm install --frozen-lockfile
pnpm demo:seed
pnpm demo:agent-run
pnpm dev -- --hostname 127.0.0.1 --port 3100
```

Open in your browser:

- http://127.0.0.1:3100/                    — homepage
- http://127.0.0.1:3100/signals             — signal inbox
- http://127.0.0.1:3100/signals/signal_mantle_spcxx_demo01  — sample detail

## What is mocked / demo vs real

| Layer | Source of truth |
|---|---|
| Demo signals | Generated by `src/demo/demoEvents.ts` (deterministic IDs, no external network). |
| Mantle docs citation | Real Mantle docs **only** when the optional `mantle-agent-skill` is present. Otherwise the skill returns `MCP_UNAVAILABLE` and the demo continues without citations. See "Mantle MCP dependency" below. |
| Fast DEX import | Reads a JSONL file you provide; Zod-validates each line. |
| Dashboard | Server-rendered Next.js pages; no external API calls. |
| Decision verdicts | The system records human-confirmed verdicts. It does NOT auto-execute them. |

## Mantle MCP dependency (judge-safe fallback)

The optional `mantle-agent-skill` provides real Mantle docs citations:

- Endpoint: `https://docs.mantle.xyz/network/~gitbook/mcp`
- Tools: `searchDocumentation(query)`, `getPage(url)`
- Source code: `src/skills/mantleDocsSkill.ts`

When the skill is **not installed** (the default in a clean clone):

- `searchMantleDocs()` returns `status: "MCP_UNAVAILABLE"` with a clear `warning` and `reason`.
- Demo flow, tests, build, and typecheck all continue to work.
- The README wording "Mantle docs auto-citation" should be read as:
  *real citation when the optional Hermes skill is present, otherwise the agent loop surfaces a `MCP_UNAVAILABLE` status and continues without citations.*

To enable real citations: install the `mantle-agent-skill` Hermes skill and ensure the Python client is reachable at `$HERMES_HOME/skills/web3/mantle-agent-skill/scripts/mantle_mcp_client.py`. No code changes required.

## JSONL store — single-process only

`src/store/eventStore.ts` is optimized for the single-process hackathon demo:

- Reads the existing file **at most once per batch**, not once per event.
- Dedup is done in-memory against a `Set` of eventIds.
- Output order is preserved.

**Honest limitation:** the store does NOT use OS-level file locking (no `flock`, no OFD locks, no POSIX advisory locks). Concurrent writers from separate processes can race and produce duplicate or interleaved writes. Do not run multiple `pnpm demo:*` processes in parallel against the same store without adding proper locking first. This is documented in code comments and in the README.
